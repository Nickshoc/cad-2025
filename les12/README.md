## Контрольные вопросы

1.
Spring MVC (Model–View–Controller) — это веб-фреймворк внутри экосистемы Spring, предназначенный для создания веб-приложений и REST API. Он реализует классический MVC-подход к разделению логики приложения:

Model: бизнес-данные и логика

View: представление (веб-страницы или JSON-ответы)

Controller: связывающее звено между моделью и представлением

2.
DispatcherServlet

Главный контроллер, фронт-контроллер всего фреймворка. Он перехватывает все входящие HTTP-запросы и направляет их к нужным компонентам.

3. @Controller

4. @Controller Используется, когда возвращается View.

@RestController Аннотация @RestController в Spring Framework представляет собой удобную комбинацию двух других аннотаций: @Controller и @ResponseBody. Она используется для создания RESTful веб-сервисов, где каждый метод контроллера возвращает данные, которые напрямую записываются в тело ответа (например, в формате JSON или XML), а не имя представления для отображения.

5. @PathVariable

6. 
Model, ModelMap и ModelAndView — это интерфейсы и классы, которые используются для передачи данных между контроллером и представлением в Spring MVC.

Model

Интерфейс для добавления атрибутов в модель, которые будут доступны в представлении.

Используется для передачи данных в шаблон или на страницу. 

ModelMap

Расширяет интерфейс Model, но с дополнительной функциональностью.

Работает по принципу Map (ключ-значение), где ключом является имя атрибута, а значением — данные, которые передаются в представление.

ModelMap используется для упрощения инициализации атрибутов с помощью цепочки вызовов. 

ModelAndView

Сложный объект, который содержит как данные модели, так и имя представления.

Используется для явного указания имени представления и атрибутов, которые будут переданы. 

7. @RequestMapping — это фундаментальная аннотация в Spring MVC (Model-View-Controller) и Spring WebFlux, которая используется для сопоставления HTTP-запросов с конкретными методами-обработчиками внутри контроллеров.

8.
@GetMapping: Для обработки HTTP GET-запросов.

@PostMapping: Для обработки HTTP POST-запросов.

@PutMapping: Для обработки HTTP PUT-запросов.

@DeleteMapping: Для обработки HTTP DELETE-запросов.

@PatchMapping: Для обработки HTTP PATCH-запросов.

9. ViewResolver в Spring MVC — это интерфейс, который используется для разрешения и определения представлений (views) в приложении.

ViewResolver нужен для того, чтобы определять, какое представление необходимо отобразить для определённого запроса.

10. В Spring MVC для возвращения строки в формате JSON из контроллера RestController нужно применить аннотации @RestController и @GetMapping.

Контрольные вопросы:

1. Servlet — это Java-класс, предназначенный для обработки HTTP-запросов и генерации HTTP-ответов. Они лежат в основе традиционного подхода к созданию веб-приложений на Java и являются частью спецификации Java EE (сейчас Jakarta EE).

2. Файл web.xml (дескриптор развёртывания) — это конфигурационный файл для веб-приложений на Java, который определяет настройки сервлетов, фильтров, слушателей и других компонентов.

3. WAR-файл (Web Application Archive) — это формат файла, который используется для упаковки и распространения веб-приложений на платформе Java EE. Он содержит файлы .jsp, сервлеты, классы Java, XML, библиотеки Java, статические веб-страницы и другие ресурсы, необходимые для работы веб-приложения. 

JAR-файл (Java Archive) — это формат файла, который содержит файлы классов, ресурсы, зависимые библиотеки и другие необходимые для приложения файлы. Основное назначение JAR-файлов — это упаковка и распространение Java-приложений или библиотек, которые могут быть запущены на любой машине, где установлена Java.

4. ServletContext — это интерфейс в Java Servlet API, который представляет собой контекст веб-приложения и предоставляет методы для взаимодействия с контейнером сервлетов

5. HttpServletRequest содержит запрос, поступивший от клиента. Его методы облегчают доступ к данным запроса. 

HttpServletResponse, в свою очередь, определяет ответ клиенту. Его методы помогают сервлету сформировать ответ.

6. Нужно реализовать интерфейс jakarta.servlet.ServletContextListener

7. Чтобы получить доступ к Spring ApplicationContext внутри обычного сервлета, можно использовать вспомогательный класс WebApplicationContextUtils из Spring.

8. ContextLoaderListener в Spring-приложении отвечает за загрузку контекста Spring (ApplicationContext) при запуске веб-приложения.

9. Использование @WebServlet позволяет заменить разделы servlet и servlet-mapping из файла web.xml. Аннотацию применяют, когда нужно реализовать веб-сервлет без необходимости создавать XML-конфигурацию.  43

Конфигурация в web.xml позволяет описывать различные параметры приложения, такие как:

Декларацию сервлетов и маппингов для них. Можно передать параметры, специфичные для конкретных сервлетов.

Фильтры, слушатели и их параметры. От правил применения (например, фильтры только для конкретных путей) до порядка применения.

Поведение по умолчанию. Например, установка сервлета, который обрабатывает запросы, если нет подходящего маппинга. Или страницу, которую сервер будет отдавать при возникновении ошибок.

Настройки авторизации и безопасности. От ограничения доступа к конкретным ресурсам сервера до конфигурации сессии (например, её времени жизни).

Выбор между @WebServlet и конфигурацией в web.xml зависит от потребностей разработчика. Если важно хранить всю конфигурацию в одном месте, то лучше использовать web.xml. Если же файлы конфигурации XML становятся слишком тяжёлыми из-за большого количества объявлений и элементов конфигурации, то стоит применять аннотации. 

10. Несколько способов использовать один Spring Bean в нескольких сервлетах:

Использовать ContextLoaderListener. Создать родительский контекст, в котором сервлеты будут дочерними элементами, и определить компоненты в родительском контексте.

Загрузить XML-файл другого сервлета вручную. 

Импортировать компоненты сервлета в другой сервлет. Для этого можно использовать, например, такой элемент: <ресурс импорта="classpath:application-context.xml"/>.

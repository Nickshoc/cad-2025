## Контрольные вопросы

1.

Spring Security является основным инструментом реализации безопасности на уровне приложений в Spring. Как правило, его цель – предложить вам настраиваемый способ реализации аутентификации, авторизации и защиты от распространенных атак. Spring Security – это программное обеспечение с открытым исходным кодом, выпущенное по лицензии Apache 2.0.

2.

Authentication — процесс проверки личности пользователя

Пример: проверка логина/пароля в форме

Результатом является Authentication-объект

Authorization — проверка, имеет ли пользователь доступ к ресурсу

Пример: доступ к /admin только для пользователей с ролью ADMIN

3. 

SecurityFilterChain — это интерфейс в Spring Security, который определяет последовательность фильтров, которые будут применяться к каждому HTTP-запросу, поступающему в приложение.

4.

Form-based аутентификация в Spring Security позволяет проводить аутентификацию пользователей с помощью настраиваемой формы входа вместо стандартного запроса безопасности.

5. 

UserDetailsService — это компонент механизма аутентификации Spring Security, который отвечает за получение информации о пользователях во время процесса аутентификации.

7. 

Basic Authentication — это метод аутентификации доступа к ресурсам по HTTP, при котором имя пользователя и пароль передаются в заголовке запроса. Эти данные кодируются с использованием Base64. 

Использовать Basic Authentication удобно в следующих случаях:

Внутренние системы. Для внутренних API или панелей управления, где важна простота, а HTTPS уже настроен. 

Быстрое прототипирование. При создании прототипов или тестировании систем Basic Authentication может сэкономить время перед реализацией более надёжного решения. 

Простая интеграция. Для простых приложений или микросервисов, где требуется ограниченный контроль доступа.

8. 

Чтобы запретить доступ к URL-адресу без соответствующей роли в Spring Security, можно использовать метод authorizeRequests() для указания правил авторизации запросов.  

Один из способов запретить доступ ко всем запросам — использовать anyRequest().denyAll(). В результате любой URL станет недоступным, и при запросе приложения будет выдаваться ответ 403 Forbidden
